Python 异步编程实战：用 asyncio 写出高性能代码

📅 2026-05-06 📂 Python编程
Python编程 Python asyncio

为什么需要异步编程？

你有没有遇到过这种情况：你的程序在等待网络请求时卡住了，CPU 却在旁边「摸鱼」？

import requests
import time
同步方式：一个一个地请求

def fetch_urls():
    start = time.time()
    
    urls = [
        "https://api.github.com",
        "https://jsonplaceholder.typicode.com/posts",
        "https://httpbin.org/get",
    ]
    
    for url in urls:
        resp = requests.get(url)
        print(f"获取到 {len(resp.text)} 字节")
    
    print(f"总耗时: {time.time() - start:.2f}秒")
fetch_urls()

输出: 总耗时: 3.21秒

这个程序大部分时间都在等网络响应，CPU 利用率极低。异步编程就是来解决这个问题的——在等待的时候去做别的事。

async/await 核心概念

协程（Coroutine）

协程是可以暂停和恢复的函数。用 async def 定义：

async def hello():
    print("你好")
    await asyncio.sleep(1)  # 暂停1秒
    print("世界")

await 关键字

await 告诉 Python：「这里要等，但你先去做别的事」。

import asyncio
async def fetch_data(url):
    print(f"开始请求: {url}")
    await asyncio.sleep(1)  # 模拟网络延迟
    print(f"完成: {url}")
    return f"数据来自 {url}"
async def main():
    result = await fetch_data("https://example.com")
    print(result)
asyncio.run(main())

并发执行

真正的高性能来自 asyncio.gather()——同时执行多个协程：

import asyncio
import time
async def fetch_data(url, delay):
    print(f"📡 请求: {url}")
    await asyncio.sleep(delay)  # 模拟不同延迟
    print(f"✅ 完成: {url}")
    return f"数据({delay}s)"
async def main():
    start = time.time()
    
    # 并发执行三个请求
    results = await asyncio.gather(
        fetch_data("URL-1", 2),
        fetch_data("URL-2", 3),
        fetch_data("URL-3", 1),
    )
    
    print(f"\n结果: {results}")
    print(f"总耗时: {time.time() - start:.2f}秒")
asyncio.run(main())

输出： 

📡 请求: URL-1
📡 请求: URL-2
📡 请求: URL-3
✅ 完成: URL-3(1s)
✅ 完成: URL-1(2s)
✅ 完成: URL-2(3s)
结果: ['数据(2s)', '数据(3s)', '数据(1s)']
总耗时: 3.01秒

三个任务同时开始，总耗时 = 最慢的那个（3秒），而不是加起来的总和（6秒）。

实战案例：异步爬虫

安装依赖

pip install aiohttp

代码实现

import asyncio
import aiohttp
import time
async def fetch_one(session, url):
    """异步请求单个 URL"""
    try:
        async with session.get(url, timeout=10) as response:
            content = await response.text()
            return url, len(content), response.status
    except Exception as e:
        return url, 0, str(e)
async def fetch_all(urls, max_concurrent=5):
    """并发请求多个 URL，限制并发数"""
    semaphore = asyncio.Semaphore(max_concurrent)
    
    async def bounded_fetch(url):
        async with semaphore:  # 控制并发数
            return await fetch_one(session, url)
    
    async with aiohttp.ClientSession() as session:
        tasks = [bounded_fetch(url) for url in urls]
        results = await asyncio.gather(*tasks)
        return results
async def main():
    urls = [
        "https://httpbin.org/delay/1",
        "https://httpbin.org/delay/2",
        "https://httpbin.org/delay/3",
        "https://httpbin.org/get",
        "https://httpbin.org/headers",
    ] * 5  # 25个请求
    
    print(f"开始并发请求 {len(urls)} 个 URL...")
    start = time.time()
    
    results = await fetch_all(urls, max_concurrent=10)
    
    elapsed = time.time() - start
    success = sum(1 for r in results if r[2] == 200)
    
    print(f"\n✅ 成功: {success}/{len(results)}")
    print(f"⏱️  耗时: {elapsed:.2f}秒")
    print(f"🚀 速率: {len(results)/elapsed:.1f} 请求/秒")
asyncio.run(main())

常见陷阱

❌ 陷阱 1：在协程中使用阻塞调用

# 错误！time.sleep() 是阻塞的，会卡住整个事件循环
async def bad():
    time.sleep(1)  # 会阻塞
    return "hi"
正确

async def good():
    await asyncio.sleep(1)  # 不会阻塞
    return "hi"

❌ 陷阱 2：忘记 await

async def wrong():
    result = fetch_data()  # 返回的是协程对象，不是数据
    return result
async def right():
    result = await fetch_data()  # 正确的等待
    return result

❌ 陷阱 3：同步库混用

requests 不支持异步。在网络密集型的异步代码中，请使用：

  • aiohttp → 替代 requests
  • aiomysql / asyncpg → 替代 pymysql
  • aiofiles → 替代文件操作

性能对比

| 方式 | 10个请求 | 50个请求 | 100个请求 | |------|---------|---------|---------| | 同步 (串行) | 10秒 | 50秒 | 100秒 | | 多线程 | 1.5秒 | 4秒 | 8秒 | | 异步 asyncio | 1.1秒 | 2.5秒 | 4秒 |

异步在网络 I/O 场景下，是最高效的方案。

总结

  • async/await = Python 的异步编程语法糖
  • asyncio.gather() = 并发执行多个任务
  • Semaphore = 控制并发数，避免被目标服务器封 IP
  • aiohttp = 异步 HTTP 请求库

    • 异步编程的适用场景：

    • ✅ 网络请求（API调用、爬虫）
    • ✅ 文件读写
    • ✅ 数据库操作
    • ❌ CPU 密集型任务（用多进程）
    掌握 asyncio，你的 Python 代码会在 I/O 密集场景下快 5-10 倍。
