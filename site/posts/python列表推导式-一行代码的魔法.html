Python列表推导式：一行代码的魔法

在Python开发中，高效处理数据是关键。列表推导式作为一种简洁的语法，能用一行代码完成复杂列表操作，让代码更优雅、可读性更高。本文将带你深入实践，掌握这一高效工具在真实场景中的应用。

一、列表推导式的本质：从冗长循环到优雅表达

列表推导式是Python中一种通过for循环和条件判断直接生成列表的语法结构。它本质上将传统for循环和if条件的组合压缩成一行代码，避免了中间变量和临时列表的冗余操作。对于熟悉Python的开发者，理解其核心是“用表达式代替循环体”。

# 传统方式：生成1-10的平方列表
squares = []
for x in range(1, 11):
    squares.append(x ** 2)
列表推导式：一行代码

squares = [x ** 2 for x in range(1, 11)]

应用场景：在数据预处理中，快速生成数值序列。例如，对传感器采集的原始数据进行平方变换，用于后续统计分析。运行上述代码可直接得到[1, 4, 9, 16, 25, 36, 49, 64, 81, 100]。

二、基础语法实战：过滤与映射的无缝融合

列表推导式支持在单行中同时完成数据过滤和转换。核心语法结构为[表达式 for 变量 in 可迭代对象 if 条件]。通过if条件实现筛选，表达式定义输出值，这种组合能高效处理数据清洗任务。

# 传统方式：从1-10中筛选偶数并乘以2
even_doubled = []
for x in range(1, 11):
    if x % 2 == 0:
        even_doubled.append(x * 2)
列表推导式：一行代码

even_doubled = [x * 2 for x in range(1, 11) if x % 2 == 0]

应用场景：电商系统中处理用户订单。假设从API获取的订单列表中，需快速过滤出金额大于100元且状态为“已支付”的订单，并计算折扣价（如原价×0.9）。此推导式可直接嵌入业务逻辑，提升响应速度。

三、进阶技巧：嵌套循环与多条件处理

当数据结构复杂时，列表推导式支持嵌套循环和多级条件判断。例如，处理二维数据时，通过两层循环实现筛选，避免多层for循环的冗余。关键在于保持逻辑清晰，避免过度嵌套。

# 2D数据示例：订单列表（订单ID、金额、状态）
orders = [
    {"id": 1, "amount": 80, "status": "paid"},
    {"id": 2, "amount": 120, "status": "pending"},
    {"id": 3, "amount": 50, "status": "paid"}
]
传统方式：多层循环筛选

filtered_orders = []
for order in orders:
    if order["status"] == "paid" and order["amount"] > 100:
        filtered_orders.append(order["amount"] * 0.9)
列表推导式：一行代码

filtered_orders = [order["amount"] * 0.9 for order in orders if order["status"] == "paid" and order["amount"] > 100]
